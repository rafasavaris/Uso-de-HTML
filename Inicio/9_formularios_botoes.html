Formulários e Botões em HTML

Em páginas e aplicações web, formulários e botões são essenciais para a interação do usuário. Eles permitem que os usuários enviem dados, realizem ações e interajam com o conteúdo de forma dinâmica através de campos de entrada.

Formulário

Um formulário é um agrupamento de campos de entrada que permitem que o usuáreio insira dados. Os formulários são criados com a tag <form> e podem conter diveros tipos de campos, como texto, senha, checkboxes, etc. Essa tag por si só apenas destaca um formulário na página, mas não realiza nenhuma ação. Assim, para que o formulário funcione, é necessário adicionar os campos de entrada e botões apropriados (sendo o botão de "submit" essencial para o nevio de dados).

Atributos da tag <form>

A tag <form> pode conter diversos atributos que definem seu comportamento e aparência. Alguns dos mais comuns são:

  • action - define para onde (qual servidor) os dados do formulário serão enviados quando o usuário clicar um botão de envio.
  • method - especifica o método HTTP a ser usado ao enviar os dados do formulário. Os métodos mais comuns são GET e POST. O método GET envia os dados como parte da URL (ficam visíveis), sendo o método padrão, enquanto o POST envia os dados no corpo da requisição, tornando-o mais seguro.

Campos de entrada

Os campos de entradas são elementos dentro do formulário que permitem ao usuário inserir dados. A criação de campos de entrada é, em geral, feita com a tag <input>, que pode ter diferentes tipos, dependendo do atributo type. Os tipos são:

  • text - textos em geral;
  • email - textos no formato de e-mail;
  • number - apenas números;
  • password - representa uma senha, aceitando textos em geral mas com o conteúdo escondido;
  • hidden - um campo de texto que não aparece na página;
  • file - campo para seleção de arquivo;
  • date - calendário para a escolha de uma data;
  • radio - escolha de um único valor de um grupo;
  • checkbox - escolha de valores de um grupo;
  • color - escolha de uma cor;
  • time - escolha um horário (sem fuso);
  • url - urls;
  • tel - telefone;
  • search - campo de pesquisa;
  • submit/reset/button - tipos de botões.

Eles serão abordados com mais detalhes em sequência. Há também outros tipos que são "variações" de outros, como datatime-local, month, week (variações de date) e range (variação de input). É também importante mencionar que a tag <input> não possui uma tag de fechamento, ou seja, ela é uma tag auto-fechante. Além disso, é importante definir o atributo name, o qual define um nome para o campo de entrada. É com esse nome que o servidor irá identificar os dados enviados pelo formulário.

Além disso, é possível ter outros tipos de campos de entrada, sem o uso de <input>. Os principais são:

  • <select> - seleção de um valor de uma lista, em que seu aspecto visual é um campo de texto com uma seta para baixo;
  • <textarea> - escrita de textos maiores;

Label

A tag <label> é usada para associar um rótulo a um campo de entrada. Isso melhora a acessibilidade e a usabilidade do formulário, pois permite que os usuários cliquem no rótulo para focar no campo de entrada correspondente. Para associar um rótulo a um campo de entrada, usamos o atributo for na tag <label> com o valor igual ao atributo id do campo de entrada.
Um exemplo dessa tag é: <label for="id_input"> Texto do label</label>

Button

A tag <button>, que não é autofechante, é utilizada para criar botões em formulários ou em outras partes da página. Ela pode ser usada para enviar formulários (estando dentro da tag <form>), redefinir campos ou executar ações personalizadas com JavaScript. Existem três tipos principais de botões, definidos pelo atributo type:

  • submit: envia o formulário ao servidor (padrão quando dentro de um formulário);
  • reset: redefine todos os campos do formulário para seus valores iniciais;
  • button: não realiza nenhuma ação por padrão, sendo usado para scripts personalizados.

Um exemplo de uso é:
<button type="submit">Aperte aqui para enviar!</button>
<button type="reset">Aperte aqui para resetar!</button>
<button type="button">Botão sem função.</button>

Diferença entre <button> e <input> dos tipos submit, reset e button

A principal diferença entre usar a tag button e input dos tipos submit, reset ou button está na diferença que button aceita HTML interno. Isso significa que é possível adicionar ícones nos botões, utilizar a tag span, entre outras personalizações. Assim, se deseja-se um estilo personalizado para o botão, o uso de <button> é melhor.

Type Range

O tipo de campo de entrada range é usado para selecionar um valor numérico dentro de um intervalo. Ele exibe um controle deslizante que permite ao usuário escolher um valor entre um mínimo e um máximo, ou seja, sua diferença principal para o tipo number é a forma visual. Para criar um campo de entrada do tipo range, usamos a tag:
<input type="range" name="intervalo" id="input_range">
dentro do formulário, associando também a um rótulo. Cabe ressaltar que o valor escolhido apenas irá aparecer na tela se for determinado em JavaScript.

Além disso, é possível, também, definir os atributos min, max e step, como no exemplo abaixo:
<input type="range" name="intervalo" id="input_range" min="0" max="100" step="2">
Em que o valor mínimo é 0, o máximo é 100 e o incremento é de 2. Apesar dessas configurações serem feitas, não é efeito visual prático, pois não há nenhum script feito para mostrar o valor selecionado.

Type Radio

O tipo de campo de entrada Radio Button solicita que o usuário escolha um valor dentre um grupo de opções disponíveis. Cada opção do tipo Radio é definido por um input do tipo radio, e todos os inputs do mesmo grupo devem ter o mesmo atributo nome, sendo que APENAS UMA opção pode ser selecionada. Normalmente, cada Radio Button é associado a um rótulo para identificar a opção. Além disso, nesse tipo de campo de entrada, o atributo value é essencial, pois define o valor enviado ao servidor quando o formulário é enviado. Se ele não for especificado, é enviado "on". Assim, uma forma de criar radio buttons é:
<input type="radio" name="opcoes" id="opcao1" value="1" checked>
<label for="opcao1">Opção 1</label>
<input type="radio" name="opcoes" id="opcao2" value="2">
<label for="opcao2">Opção 2</label>

Type Checkbox

Os inputs de tipo checkbox permitem a seleção de valores dentro de um grupo. A diferença dele para radio é que o usuário pode selecionar múltiplas opções, ou nenhuma. Assim, para opções do mesmo grupo, cada input deve ter o mesmo atributo name, mas cada um deve ter um valor diferente no atributo value. Além disso, cada input deve ter um rótulo associado. Um exemplo de como criar checkboxes é:
<input type="checkbox" name="grupo" id="opcao1_checkbox" value="1">
<label for="opcao1">Opção 1</label>
<input type="checkbox" name="grupo" id="opcao2_checkbox" value="2">
<label for="opcao2">Opção 2</label>

Atributos específicos do tipo Checkbox

Tipos de botões

Type Sumbit/Reset/Button

Apesar de ser possível adicionar botões dentro dos formulários com a tag <button>, também é possível fazer isso com a tag <input>. Para isso, no atributo type pode ser posto:

  • submit - botão para enviar o formulário;
  • reset - botão para limpar o formulário;
  • button - botão sem função.

Para escrever algo dentro do botão, utiliza-se o atributo value. Dessa forma, é possível criar botões assim, dentro da tag <form> como mostrado abaixo:
<input type="submit" value="Aperte aqui para enviar!">
<input type="reset" name="Limpar!">
<input type="button" value="Botão genérico!">

Type Image

Outro tipo de botão mais costumizável com o input, é o type image. Ele é, em resumo, uma imagem cliclável que permite que o usuário envie o formulário quando pressionado. Assim, ele funciona da mesma forma que um botão do tipo submit e precisa ter o atributo src, isto é, o caminho da imagem. Também é possível colocar outros atributos comuns de imagens nessa tag. Assim, essa tag é usada dentro de um <form> da seguinte forma:

Outros tipos de entrada

Select

A tag <select> é utilizada para criar uma lista suspensa (dropdown) em formulários, permitindo que o usuário escolha uma opção entre várias disponíveis. Cada opção é definida pela tag <option> dentro do <select>. A sintaxe básica é:
<select name="opcao" id="opcao">
<option value="1">Opção 1</option>
<option value="2" select>Opção 2</option>
<option value="3">Opção 3</option>
</select>
Principais atributos da tag <option>:

  • value: valor enviado ao servidor ao selecionar a opção;
  • selected: deixa a opção pré-selecionada.

Text Area

A tag <textarea> é utilizada para criar campos de texto multilinha em formulários, permitindo que o usuário digite textos longos, como comentários, descrições ou mensagens. Diferente do <input type="text">, que é limitado a uma única linha, o <textarea> pode ser redimensionado pelo usuário (dependendo do navegador) e exibe várias linhas por padrão. O texto inserido entre as tags <textarea> e </textarea> será exibido como valor padrão do campo. A sintaxe básica é: <textarea name="mensagem" id="mensagem" rows="4" cols="50"></textarea> Os principais atributos são:

  • rows: define o número de linhas visíveis;
  • cols: define o número de colunas (largura);
  • maxlength: limita o número máximo de caracteres.

Atributos gerais

  • placeholder - define uma dica do que o campo espera (aceito em campos de escrita);
  • required - indica um campo obrigatório para o envio do formulário (aceito em todos os campos);
  • readonly - indica que o campo não pode ser alterado, pois é apenas para leitura (aceito em campos de escrita e tempo);
  • disabled - indica que o campo não pode ser alterado nem receber foco (aceito em todos os campos);
  • value - define o valor inicial do campo (aceito em praticamente todos os campos de input);
  • name - determina o nome do campo no formulário, sendo útil quando ele é enviado para o servidor (aceito em todos os campos);
  • maxlength - determina o número máximo de caracteres de um <input> (aceito em campos de escrita de texto).
